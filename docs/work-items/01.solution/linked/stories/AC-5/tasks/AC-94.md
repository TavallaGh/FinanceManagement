## 2. Description

Refactor the audit logging domain model from the current user-targeted structure to a scalable, extensible, entity-agnostic design. This task updates the `AuditLog` entity, related enums, EF Core mappings, and database schema so audit records are reusable across entity types rather than tied to a narrow user-focused shape.

## 3. Goal Of Task

Migrate the audit logging domain model to a scalable and extensible design that supports generic audit actions, actor classification, request metadata, and correlation tracking while removing legacy audit coupling and keeping the solution buildable and migration-ready.

## 4. What Problem This Task Should Be Done For

The current audit model is tightly coupled to legacy fields such as `TargetUserId`, `TargetUserName`, and `ActorSubject`, plus a concrete `AuditLogService` implementation that is embedded in current workflows. That design blocks reuse across entities, makes system-generated and service-generated events awkward, leaves stale foreign keys and indexes in the schema, and increases the cost of future audit expansion.

## 6. Scope

- In Scope:
  - `AuditLog` entity redesign.
  - New audit actor and target classification enums.
  - EF Core mapping and schema delta for the refactored audit model.
  - Removal of legacy audit columns, foreign keys, indexes, service implementation, DTO/contract usage, and dependent references.
  - Build, migration, persistence, and repo-search validation for the audit refactor.
- Out Of Scope:
  - Implementing a replacement `AuditLogService`.
  - Producing audit events from business workflows.
  - Integrating the new audit model into application operations.
  - Any frontend or API behavior changes outside compile-safe dependency cleanup.
