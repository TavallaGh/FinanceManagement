## 2. Description

Implement `UiListWithPaginationComponent` as an Angular standalone component in `libs/shared/ui/src/lib/components/list-with-pagination/`. The component is a stateless, data-driven list container with integrated pagination controls. It accepts page size, current page index, and total item count as inputs and emits page change events — all pagination state is owned by the consumer. List item content is projected via `ng-content`. All styling is token-driven, theme-aware, and RTL/LTR-safe. Export from `index.ts`. Add a Storybook page at `story-book/pages/list-with-pagination/`. Register the route.

## 3. Goal Of Task

Provide a stateless, reusable paginated list container so every ERP feature that needs paginated data uses the same pagination UX and controls — without reimplementing page navigation logic or controls per feature.

## 4. What Problem This Task Should Be Done For

Paginated data lists are currently implemented independently per feature — each with its own pagination controls, page state management convention, and visual treatment. This creates inconsistent UX and forces each feature to solve the same pagination interaction problem from scratch.

## 5. AoC (Acceptance of Completion)

- AOC-01: `UiListWithPaginationComponent` implemented with inputs: `pageSize: number`, `pageIndex: number`, `totalItems: number`; and output: `pageChange: EventEmitter<{ pageIndex: number, pageSize: number }>`.
- AOC-02: Pagination controls render: current page indicator, total pages indicator, previous page button, next page button.
- AOC-03: Previous button is disabled on the first page; next button is disabled on the last page.
- AOC-04: `pageChange` event emits with new `pageIndex` when previous or next is clicked; component does not manage internal page state (stateless — consumer drives state).
- AOC-05: List content area renders projected content via `ng-content`.
- AOC-06: Configurable `emptyMessage` input renders when `totalItems=0`.
- AOC-07: All styling (pagination bar, buttons, indicators) uses semantic tokens exclusively — zero hardcoded values.
- AOC-08: Pagination controls use logical CSS properties for RTL/LTR safety.
- AOC-09: Component uses `ChangeDetectionStrategy.OnPush` and standalone pattern.
- AOC-10: Storybook page at `story-book/pages/list-with-pagination/` covers: first page, last page (next disabled), middle page, single page (both disabled), empty state, RTL layout.
- AOC-11: Component exported from `libs/shared/ui/src/lib/components/index.ts`.
- AOC-12: Route registered in `story-book.routes.ts`; navigation entry in Storybook shell sidebar.
- AOC-13: Translation keys for pagination labels (`en.json` and `fa.json`): page indicators, previous/next labels.
