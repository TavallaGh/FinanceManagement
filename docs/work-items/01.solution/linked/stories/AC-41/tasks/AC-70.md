- Stack: Frontend / Styles

## 2. Description

Deliver a shared responsive layout primitive for the ERP frontend as a CSS utility class set — not an Angular component. The classes provide column-based grid and flexbox row layout driven by shared spacing tokens from TBD-01. All gap and padding values are resolved from semantic spacing tokens. Classes use logical CSS properties for RTL/LTR safety. Document the class API in `docs/frontend/design/design-system.md`. Add Storybook examples under `story-book/pages/grid/` demonstrating responsive column layouts and RTL behavior.

## 3. Goal Of Task

Provide a shared, token-driven layout primitive so feature teams stop rolling their own grid/flexbox with hardcoded gaps and duplicate spacing values.

## 4. What Problem This Task Should Be Done For

Feature teams have no shared layout primitive. Each feature implements its own grid/flexbox layout with hardcoded gap values — creating visual inconsistency and making global spacing changes require edits across every feature layout file.

## 5. AoC (Acceptance of Completion)

- AOC-01: CSS utility class set delivered (e.g., `ui-grid`, `ui-grid--cols-{n}`, `ui-row`, `ui-col`, `ui-gap-{size}`) covering 1–12 column configurations.
- AOC-02: All gap and gutter values resolve from semantic spacing tokens — zero hardcoded pixel values.
- AOC-03: Classes use logical CSS properties (`gap`, `padding-inline`, `margin-inline`) for RTL/LTR correctness.
- AOC-04: Responsive breakpoint modifiers provided for common breakpoints (sm, md, lg) using token-aligned breakpoint values.
- AOC-05: Grid class API documented in `docs/frontend/design/design-system.md` with usage examples and token references.
- AOC-06: Storybook page at `story-book/pages/grid/` demonstrates: single-column, multi-column (2, 3, 4, 12), responsive breakpoint behavior, and RTL layout correctness.
- AOC-07: Route registered in `story-book.routes.ts`; navigation entry in Storybook shell sidebar.
- AOC-08: No Angular component or directive is created — layout utility is CSS-only with optional lightweight wrapper elements where ergonomics require.

## 6. Scope

- In Scope:
  - `public/styles/` — grid utility class file (e.g., `_grid.scss` or `_layout.scss`)
  - Column grid and flex row layouts
  - Spacing token–driven gaps and gutters
  - Responsive breakpoint modifiers
  - Storybook page and route registration
  - `design-system.md` documentation
- Out Of Scope:
  - Angular component or directive wrapping (Decision 1 in solution.md)
  - CSS Grid subgrid (not needed for MVP)
  - Animation or transition on layout change

## 7. Expectations

- Expected outcome:
  - A developer can apply `class="ui-grid ui-grid--cols-3 ui-gap-md"` to any host element and get a properly spaced, token-driven, RTL-safe 3-column grid.
- Success signals:
  - Storybook examples render correct layouts in both LTR and RTL at multiple breakpoints.
  - Zero hardcoded pixel values in the utility class file.
