# Solution Task Spec: AC-64

## 1. Task Identity

- Parent Story: AC-41
- Jira Task/Subtask: [AC-64](https://nexttoptech.atlassian.net/browse/AC-64)
- Task Name: FE - Token Audit & Standardization
- Stack: Frontend / Styles
- Status: draft

## 2. Description

Audit all six existing token files in `public/styles/tokens/` (`_colors.scss`, `_spacing.scss`, `_typography.scss`, `_radius.scss`, `_elevation.scss`, `_z-index.scss`) and establish a clear, enforced two-layer token model. Layer 1 (primitive) holds raw scale values; Layer 2 (semantic) holds named aliases that all components must exclusively consume. Produce a new `_semantic.scss` (or per-category semantic files) and update `docs/frontend/design/design-system.md` with documented token usage rules.

## 3. Goal Of Task

Establish the single source of truth for all visual values in the shared UI system — the mandatory prerequisite before any component implementation begins.

## 4. What Problem This Task Should Be Done For

Token consumption is uncontrolled — components may reference primitive tokens directly or hardcode values, making theme and branding changes fragile and requiring surgical edits across component files instead of a single token change.

## 5. AoC (Acceptance of Completion)

- AOC-01: All six token files audited; every variable categorized as primitive or semantic with no ambiguity.
- AOC-02: A semantic token layer exists (e.g., `_semantic.scss`) defining named aliases for all categories: color (`--color-surface`, `--color-on-surface`, `--color-border`, `--color-text-primary`, etc.), spacing, typography, radius, elevation, and z-index.
- AOC-03: Primitive token files cleaned of duplicates and aligned to a consistent naming convention (`--{category}-{scale}`).
- AOC-04: Token usage rules documented in `docs/frontend/design/design-system.md`: which layer components consume, naming convention for both layers, and rationale.
- AOC-05: Violation list produced — all existing 6 shared components (`checkbox`, `checkbox-group`, `date-picker`, `icon`, `tag`, `tag-group`) checked for hardcoded values or primitive token direct references; findings documented as input to TBD-15.
- AOC-06: App compiles and existing Storybook pages render without visual regression after token file changes.

## 6. Scope

- In Scope:
  - `public/styles/tokens/` — all six files
  - Creation of semantic token layer file(s)
  - Token naming convention definition and documentation
  - Violation audit of existing 6 shared components (read-only, report only — no refactoring here)
  - Update of `docs/frontend/design/design-system.md`
- Out Of Scope:
  - Component refactoring (that is TBD-15)
  - Theme light/dark value assignment (that is TBD-02)
  - Any new component implementation

## 7. Expectations

- Expected outcome:
  - A clean, documented two-layer token system that can be consumed confidently by all subsequent component tasks.
- Success signals:
  - Any developer can look up the correct CSS variable for any visual property using only the semantic layer.
  - No new component ever needs to reference a primitive token or hardcode a visual value.
