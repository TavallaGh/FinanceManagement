# Solution Task Spec: AC-69

- Stack: Frontend / UI

## 2. Description

Implement `UiCardComponent` as an Angular standalone component in `libs/shared/ui/src/lib/components/card/`. The component provides a flexible content container with header, body, and footer regions via `ng-content` projection. Elevation and spacing are fully token-driven. The component is theme-aware, RTL/LTR-safe using logical CSS properties, and business-agnostic. Export from `index.ts`. Add a Storybook page at `story-book/pages/card/` covering composition variants. Register the route and add translation keys.

## 3. Goal Of Task

Provide a reusable content container so feature UIs have a consistent, token-driven card abstraction for wrapping sections, panels, and data blocks — eliminating ad-hoc wrappers with hardcoded shadows and spacing.

## 4. What Problem This Task Should Be Done For

Feature UIs have no shared container abstraction. Layouts use inconsistent wrapping elements with hardcoded box-shadows and spacing values — making global card styling changes require edits across every feature.

## 5. AoC (Acceptance of Completion)

- AOC-01: `UiCardComponent` implemented with three named content projection slots: `[uiCardHeader]`, `[uiCardBody]` (default), and `[uiCardFooter]`.
- AOC-02: Header and footer slots are optional — card renders correctly when only body content is projected.
- AOC-03: Elevation (box-shadow) and internal spacing (padding) are resolved exclusively through semantic elevation and spacing tokens.
- AOC-04: Component is theme-aware — elevation and background tokens resolve correctly in both light and dark themes.
- AOC-05: Component uses logical CSS properties for all directional spacing and border values.
- AOC-06: Component uses `ChangeDetectionStrategy.OnPush` and standalone pattern; zero hardcoded visual values.
- AOC-07: Storybook page at `story-book/pages/card/` covers: body-only, header+body, header+body+footer, content-rich body example, RTL layout example.
- AOC-08: Component exported from `libs/shared/ui/src/lib/components/index.ts`.
- AOC-09: Route registered in `story-book.routes.ts`; navigation entry in Storybook shell sidebar.
- AOC-10: Translation keys for Storybook demo labels added to `en.json` and `fa.json`.

## 6. Scope

- In Scope:
  - `libs/shared/ui/src/lib/components/card/` — `UiCardComponent`
  - Three content projection slots: header, body, footer
  - Token-driven elevation and spacing
  - Storybook page and route registration
- Out Of Scope:
  - Clickable/interactive card variant (not in story scope)
  - Card list or card grid layout — use Grid System (TBD-07) for layout
  - Expandable/collapsible card (not in story scope)

## 7. Expectations
