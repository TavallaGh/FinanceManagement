# Solution Task Spec: AC-66

## 1. Task Identity

- Parent Story: AC-41
- Jira Task/Subtask: [AC-66](https://nexttoptech.atlassian.net/browse/AC-66)
- Task Name: FE - Theme Persistence & No-Flicker Initialization
- Stack: Frontend / Styles
- Status: draft

## 2. Description

Persist the user's selected theme (light/dark) across page reloads and apply it before Angular bootstraps to prevent a flash of incorrect theme (FOCT). Because the project CSP `script-src` does not permit `unsafe-inline`, the standard inline `<script>` approach in `index.html` cannot be used. The solution uses a CSP-compatible strategy: a nonce-based inline script (if the server issues nonces) or an `APP_INITIALIZER`-based early-read with an accepted single-frame tradeoff. Theme preference is stored in `localStorage`; `prefers-color-scheme` is honored as default when no preference is stored.

## 3. Goal Of Task

Ensure the user's chosen theme is remembered and applied instantly on every page load with no visible flicker — while keeping the application's Content Security Policy intact.

## 4. What Problem This Task Should Be Done For

Without persistence, every page reload resets the theme to default. Without no-flicker initialization, the app briefly renders the wrong theme before Angular bootstraps — a jarring visual transition that CSS alone cannot prevent after the fact.

## 5. AoC (Acceptance of Completion)

- AOC-01: User's selected theme persists in `localStorage` across page reloads.
- AOC-02: On initial page load, the correct theme is applied before visible content renders — or, if CSP nonce is unavailable, the `APP_INITIALIZER` strategy is used and the single-frame fallback is documented as the accepted tradeoff.
- AOC-03: When no stored preference exists, the app reads `prefers-color-scheme` and defaults to the OS-level theme.
- AOC-04: Theme persistence and initialization logic live inside `ThemeService` (extending TBD-02) — no duplication of theme logic.
- AOC-05: Zero CSP violations introduced — `Content-Security-Policy` headers remain fully intact after this change.
- AOC-06: After a user sets a preference, hard refresh of the page delivers the selected theme with no observable flash.

## 6. Scope

- In Scope:
  - `localStorage` theme persistence
  - Early theme read: nonce-based inline script or `APP_INITIALIZER` strategy
  - `prefers-color-scheme` fallback
  - Extension of `ThemeService` from TBD-02
  - CSP compliance verification
- Out Of Scope:
  - Theme engine itself — light/dark token values (TBD-02)
  - Any UI toggle component (business-layer concern, not shared UI)
  - Server infrastructure changes beyond confirming nonce availability

## 7. Expectations

- Expected outcome:
  - A user who selects dark mode sees dark mode on every subsequent page load with no flash of light mode.
- Success signals:
  - Hard refresh of the page with a stored preference delivers the correct theme with no observable flash.
  - Browser DevTools shows zero CSP violations.
