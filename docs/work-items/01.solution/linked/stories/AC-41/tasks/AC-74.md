# Solution Task Spec: AC-74

## 1. Task Identity

- Parent Story: AC-41
- Jira Task/Subtask: [AC-74](https://nexttoptech.atlassian.net/browse/AC-74)
- Task Name: FE - Sidebar Component
- Stack: Frontend / UI
- Status: draft

## 2. Description

Implement `UiSidebarComponent` as an Angular standalone component in `libs/shared/ui/src/lib/components/sidebar/`. The component provides a reusable navigation container with collapse/expand behavior. Navigation item content is projected via `ng-content` or a structured slot API. The component is theme-aware (background, border, icon colors adapt to light/dark theme via tokens), direction-aware (RTL/LTR: sidebar position and text/icon alignment adapt to document direction via logical CSS), and token-driven throughout. Export from `index.ts`. Add a Storybook page at `story-book/pages/sidebar/`. Register the route.

## 3. Goal Of Task

Provide a shared navigation container so the ERP app shell can implement a sidebar layout once — using the shared component — rather than building its own collapse/expand container with state management and directional styling from scratch.

## 4. What Problem This Task Should Be Done For

No shared navigation container exists. The app shell must implement its own sidebar, including collapse/expand behavior, theme-responsive colors, and RTL/LTR directional adaptation. Without a shared component, any app that needs a sidebar duplicates this complexity.

## 5. AoC (Acceptance of Completion)

- AOC-01: `UiSidebarComponent` implemented with `collapsed` input (boolean) and `collapsedChange` output (EventEmitter<boolean>) for two-way binding.
- AOC-02: When `collapsed=true`, the sidebar renders in a narrow/icon-only state; when `collapsed=false`, it renders full-width with labels.
- AOC-03: A toggle button or control within the component toggles the collapsed state and emits `collapsedChange`.
- AOC-04: Navigation item content is rendered via `ng-content` — the component imposes no domain-specific item structure.
- AOC-05: Background color, border color, and icon/text color all resolve from semantic tokens and adapt correctly in both light and dark themes.
- AOC-06: Direction-aware: sidebar placement and item padding/margin use logical CSS properties (`padding-inline-start`, `border-inline-end`); the component renders correctly in both LTR and RTL without additional consumer CSS.
- AOC-07: Component uses `ChangeDetectionStrategy.OnPush` and standalone pattern; zero hardcoded visual values.
- AOC-08: Storybook page at `story-book/pages/sidebar/` covers: expanded state, collapsed state, with navigation items projected, light theme, dark theme, LTR layout, RTL layout.
- AOC-09: Component exported from `libs/shared/ui/src/lib/components/index.ts`.
- AOC-10: Route registered in `story-book.routes.ts`; navigation entry in Storybook shell sidebar.
- AOC-11: Translation keys for toggle button labels (`en.json` and `fa.json`).

## 6. Scope

- In Scope:
  - `libs/shared/ui/src/lib/components/sidebar/` — `UiSidebarComponent`
  - Collapsed/expanded state with toggle
  - Theme-aware token-driven styling
  - RTL/LTR direction-aware layout
  - `ng-content` navigation projection
  - Storybook page and route registration
- Out Of Scope:
  - Navigation item component (items are projected — no `UiNavItemComponent` in this task)
  - Active/selected navigation item state (feature-layer concern)
  - Overlay/modal sidebar variant for mobile (not in scope)
  - Resize-by-drag behavior

## 7. Expectations

- Success signals:
  - Storybook demonstrates expanded and collapsed states in both themes and LTR/RTL with correct token resolution.
