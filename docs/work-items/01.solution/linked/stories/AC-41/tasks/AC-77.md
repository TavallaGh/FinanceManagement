# Solution Task Spec: AC-77

## 1. Task Identity

- Parent Story: AC-41
- Jira Task/Subtask: [AC-77](https://nexttoptech.atlassian.net/browse/AC-77)
- Task Name: FE - Storybook Coverage & Design Docs Update
- Stack: Frontend / Docs
- Status: draft

## 2. Description

Consolidate Storybook shell registration for all 11 new shared components delivered in TBD-04 through TBD-13. Update the Storybook sidebar/shell component to include navigation entries for all new pages. Update `docs/frontend/design/design-system.md` to reflect finalized token rules and theme guidance. Update `docs/frontend/design/ui-components.md` to reflect the complete new shared component inventory with capabilities and Storybook route references.

Note: individual Storybook pages and translation keys are delivered within each component task (TBD-04 through TBD-13). This task handles shell-level consolidation and documentation finalization only.

## 3. Goal Of Task

Ensure the Storybook developer tool is fully navigable across all 11 new components and that design documentation accurately reflects the delivered token system, theme engine, and component inventory — so both developers and designers have a single source of truth.

## 4. What Problem This Task Should Be Done For

After 11 component tasks complete, the Storybook sidebar/shell may still only list components that existed before this story. Design documentation is outdated. Without this consolidation, developers cannot discover new components through Storybook navigation, and designers cannot verify the implementation against the documented design system.

## 5. AoC (Acceptance of Completion)

- AOC-01: Storybook sidebar/shell component (`story-book-shell.component.ts`) includes navigation entries for all 11 new component pages: input, textarea, button, card, grid, table, simple-list, list-with-pagination, sidebar, tree-view, scroll-container.
- AOC-02: All 11 new Storybook routes are verified as accessible and rendering correctly from the Storybook shell navigation.
- AOC-03: `docs/frontend/design/design-system.md` updated to reflect: finalized two-layer token model (TBD-01), light/dark theme token mapping summary, CSP-safe theme persistence strategy, and Angular Material bridge overview.
- AOC-04: `docs/frontend/design/ui-components.md` updated to include all 11 new components with: component name, selector, key inputs/outputs, Storybook route, and status (delivered).
- AOC-05: The existing 6 shared components are also present and up to date in `ui-components.md` — no entries removed.
- AOC-06: No broken Storybook routes — all registered routes resolve without errors.

## 6. Scope

- In Scope:
  - `story-book-shell.component.ts` — sidebar navigation entries for all 11 new components
  - `story-book.routes.ts` — verification that all routes from TBD-04–TBD-13 are correctly registered
  - `docs/frontend/design/design-system.md` — token and theme documentation finalization
  - `docs/frontend/design/ui-components.md` — full component inventory update
- Out Of Scope:
  - Individual component Storybook page content (each component task owns its own page)
  - Translation key additions for component demos (owned by each component task)
  - Any component implementation changes

## 7. Expectations

- Expected outcome:
  - A developer opens the Storybook shell and can navigate to any of the 17 total shared components (6 existing + 11 new) via the sidebar.
  - A developer or designer opens `ui-components.md` and finds a complete, accurate inventory of all shared components.
- Success signals:
  - All 11 new Storybook routes accessible from sidebar with no 404s.
  - Design docs reviewed and accurate.
