# Solution Task Spec: AC-76

## 1. Task Identity

- Parent Story: AC-41
- Jira Task/Subtask: [AC-76](https://nexttoptech.atlassian.net/browse/AC-76)
- Task Name: FE - Scroll Container Component
- Stack: Frontend / UI
- Status: draft

## 2. Description

Implement `UiScrollContainerComponent` as an Angular standalone component in `libs/shared/ui/src/lib/components/scroll-container/`. The component provides a reusable scroll wrapper with consistent overflow behavior and optional maintainable custom scrollbar styling. Scroll behavior and custom scrollbar treatment are defined via CSS using token-driven values where applicable. The component is theme-aware and RTL/LTR-safe. Export from `index.ts`. Add a Storybook page at `story-book/pages/scroll-container/`. Register the route.

## 3. Goal Of Task

Provide a shared scroll wrapper so scrollable regions across ERP features use consistent overflow behavior, scrollbar styling, and scroll UX — eliminating per-feature ad-hoc scroll wrappers with divergent scrollbar treatments.

## 4. What Problem This Task Should Be Done For

Scrollable regions across features have inconsistent scroll styling. Browser default scrollbars conflict with the design system's visual language, and without a shared wrapper, any custom scrollbar treatment must be reimplemented per feature — creating maintenance fragility.

## 5. AoC (Acceptance of Completion)

- AOC-01: `UiScrollContainerComponent` implemented with `orientation` input (`'vertical' | 'horizontal' | 'both'`; default `'vertical'`) controlling scroll axis.
- AOC-02: Component renders a container with correct `overflow` CSS based on `orientation` input.
- AOC-03: Custom scrollbar styling (`::-webkit-scrollbar`, `scrollbar-width`, `scrollbar-color`) applied consistently — scrollbar track and thumb colors resolve from semantic tokens.
- AOC-04: Scrollbar styling adapts to light/dark theme via token resolution.
- AOC-05: Component uses `ng-content` for projected scrollable content — no constraint on content type.
- AOC-06: Component uses logical CSS properties where applicable; scrollbar direction is RTL-compatible (scrollbar appears on the correct side in RTL documents).
- AOC-07: Component uses `ChangeDetectionStrategy.OnPush` and standalone pattern; zero hardcoded visual values.
- AOC-08: Storybook page at `story-book/pages/scroll-container/` covers: vertical scroll, horizontal scroll, both-axes scroll, themed scrollbar in light mode, themed scrollbar in dark mode, RTL layout.
- AOC-09: Component exported from `libs/shared/ui/src/lib/components/index.ts`.
- AOC-10: Route registered in `story-book.routes.ts`; navigation entry in Storybook shell sidebar.
- AOC-11: Translation keys for any Storybook demo text added to `en.json` and `fa.json`.

## 6. Scope

- In Scope:
  - `libs/shared/ui/src/lib/components/scroll-container/` — `UiScrollContainerComponent`
  - Vertical, horizontal, both scroll orientations
  - Custom scrollbar styling via CSS (`::-webkit-scrollbar` + `scrollbar-width`/`scrollbar-color` for Firefox)
  - Theme-aware scrollbar token values
  - Storybook page and route registration
- Out Of Scope:
  - Virtual/infinite scroll behavior
  - Pull-to-refresh
  - Scroll position tracking or programmatic scroll API (consumer concern)
  - Overlay/fade shadow at scroll edges (can be added as enhancement later)

## 7. Expectations

- Expected outcome:
  - Any feature wraps a content-heavy region in `<ui-scroll-container>` and gets consistent overflow behavior and a styled scrollbar that matches the design system — no custom scrollbar CSS needed in the feature.
- Success signals:
  - Storybook demonstrates custom scrollbar styling in both light and dark themes on both Chromium and Firefox rendering engines.
  - RTL scrollbar position is correct.
