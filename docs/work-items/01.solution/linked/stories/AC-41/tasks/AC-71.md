- Stack: Frontend / UI

## 2. Description

Implement `UiTableComponent` as an Angular standalone component in `libs/shared/ui/src/lib/components/table/`. The component provides a structured data table with header/body separation, row hover state, selected row state, and an empty state for when no data is present. All styling is token-driven, theme-aware, and RTL/LTR-safe. The component is data-driven via column definitions and row data inputs and remains business-agnostic. Export from `index.ts`. Add a Storybook page at `story-book/pages/table/` covering all states. Register the route.

## 3. Goal Of Task

Provide a canonical shared data table so every ERP feature that displays tabular data uses the same structure, states, and token-driven styling — eliminating per-feature ad-hoc table implementations.

## 4. What Problem This Task Should Be Done For

Data tables in ERP features are implemented ad-hoc with no shared contract for header structure, row states, or empty state presentation — creating inconsistent visual treatment and duplicated logic across every feature that renders tabular data.

## 5. AoC (Acceptance of Completion)

- AOC-01: `UiTableComponent` implemented with `columns` input (array of column definitions with `key`, `label`, optional `width`) and `rows` input (array of data objects).
- AOC-02: Table renders a `<thead>` with column labels and a `<tbody>` with rows matching the column definition.
- AOC-03: Row hover state is implemented with a token-driven hover background color.
- AOC-04: `selectedRowKey` input and `rowSelected` output implemented — clicking a row emits its key; selected row renders with a token-driven selection background.
- AOC-05: Empty state renders a configurable message (via `emptyMessage` input or content projection) when `rows` is empty or null.
- AOC-06: All states (default, hover, selected, empty) use semantic tokens exclusively — zero hardcoded values.
- AOC-07: Component uses logical CSS properties for column alignment and cell padding for RTL/LTR safety.
- AOC-08: Component uses `ChangeDetectionStrategy.OnPush` and standalone pattern.
- AOC-09: Storybook page at `story-book/pages/table/` covers: populated table, row hover (documented), selected row, empty state, RTL layout.
- AOC-10: Component exported from `libs/shared/ui/src/lib/components/index.ts`.
- AOC-11: Route registered in `story-book.routes.ts`; navigation entry in Storybook shell sidebar.
- AOC-12: Translation keys for any Storybook demo text added to `en.json` and `fa.json`.

## 6. Scope

- In Scope:
  - `libs/shared/ui/src/lib/components/table/` — `UiTableComponent`
  - Column definitions input, row data input
  - Row hover, selected row, empty state
  - Storybook page and route registration
- Out Of Scope:
  - Column sorting (not in AoC-13 scope)
  - Column filtering within the table (belongs to feature layer)
  - Virtual scrolling / infinite scroll
  - Editable cells
  - Column resizing or reordering
