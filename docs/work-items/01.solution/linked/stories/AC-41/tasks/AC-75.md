# Solution Task Spec: AC-75

## 1. Task Identity

- Parent Story: AC-41
- Jira Task/Subtask: [AC-75](https://nexttoptech.atlassian.net/browse/AC-75)
- Task Name: FE - Tree View Component
- Stack: Frontend / UI
- Status: draft

## 2. Description

Implement `UiTreeViewComponent` and `UiTreeViewNodeComponent` as Angular standalone components in `libs/shared/ui/src/lib/components/tree-view/`. The component renders hierarchical data with expand/collapse behavior per node. Data is passed as a recursive node structure. Node label content is projected or provided via data input. All styling is token-driven, theme-aware, and RTL/LTR-safe. Export from `index.ts`. Add a Storybook page at `story-book/pages/tree-view/`. Register the route.

## 3. Goal Of Task

Provide a shared tree view primitive so ERP features that need hierarchical data visualization (e.g., chart of accounts, org structure, category tree) have a single reusable implementation with consistent expand/collapse UX and token-driven styling.

## 4. What Problem This Task Should Be Done For

Hierarchical data has no shared visualization primitive. Any feature needing a tree structure must build its own recursive rendering, expand/collapse state management, and visual indentation from scratch — duplicating effort and producing divergent implementations.

## 5. AoC (Acceptance of Completion)

- AOC-01: `UiTreeViewComponent` accepts `nodes` input of type `UiTreeNode[]` where `UiTreeNode` has: `id`, `label`, optional `children: UiTreeNode[]`, optional `expanded: boolean`.
- AOC-02: Nodes with children render with an expand/collapse toggle; leaf nodes render without a toggle.
- AOC-03: Expanding a node shows its children; collapsing hides them. The component manages expand/collapse state internally or supports `expanded` as controlled input per node.
- AOC-04: `nodeSelected` output emits the selected node's `id` when a node label is clicked.
- AOC-05: Visual indentation per depth level uses spacing tokens — zero hardcoded pixel values for indentation.
- AOC-06: All styling uses semantic tokens; tree lines (if rendered) use border/color tokens.
- AOC-07: Components use logical CSS properties for indentation and icon positioning (RTL/LTR safe).
- AOC-08: Components use `ChangeDetectionStrategy.OnPush` and standalone pattern.
- AOC-09: Storybook page at `story-book/pages/tree-view/` covers: flat list (no children), single-level nesting, multi-level nesting, all-expanded, all-collapsed, node selection, RTL layout.
- AOC-10: Both components exported from `libs/shared/ui/src/lib/components/index.ts`.
- AOC-11: Route registered in `story-book.routes.ts`; navigation entry in Storybook shell sidebar.
- AOC-12: Translation keys for expand/collapse aria labels added to `en.json` and `fa.json`.

## 6. Scope

- In Scope:
  - `libs/shared/ui/src/lib/components/tree-view/` — `UiTreeViewComponent` and `UiTreeViewNodeComponent`
  - Recursive node rendering with expand/collapse
  - Node selection output
  - Token-driven indentation and styling
  - Storybook page and route registration
- Out Of Scope:
  - Drag-and-drop node reordering
  - Node editing in-place
  - Lazy loading of children from API (consumer concern)
  - Checkbox-based multi-select tree (not in AoC-17 scope)

## 7. Expectations

- Expected outcome:
  - A feature team passes `[nodes]` to `<ui-tree-view>` and handles `(nodeSelected)` to get a fully functional, styled, expandable tree — no custom recursive rendering needed.
- Success signals:
  - Multi-level nesting renders correctly with proper visual indentation.
  - Expand/collapse works at every depth level.
  - RTL indentation is mirrored correctly.

## 8. TDD Coverage

- Unit tests:
  - Leaf node renders without expand/collapse toggle.
  - Node with children renders with expand/collapse toggle.
  - Clicking toggle on a collapsed node expands it (children become visible).
  - Clicking toggle on an expanded node collapses it (children hidden).
  - `nodeSelected` emits correct `id` when node label clicked.
- Integration tests:
  - Three-level deep tree renders with correct indentation at each level.
- Contract/API tests (if applicable):
  - N/A

## 9. BDD Scenarios

- Scenario 1:
  - Given a `UiTreeViewComponent` with a three-level hierarchy, all nodes collapsed
  - When the root node's toggle is clicked
  - Then first-level children are revealed; second-level children remain hidden.
- Scenario 2:
  - Given a `UiTreeViewComponent` rendered in RTL layout
  - When inspected
  - Then indentation is applied on the inline-end side (right in LTR → left in RTL) via logical CSS.
- Scenario 3:
  - Given a node is clicked
  - When `nodeSelected` output is observed
  - Then it emits the clicked node's `id`.

- Verification checkpoints:

  - Expand/collapse unit tests pass.
  - RTL indentation verified in Storybook.
