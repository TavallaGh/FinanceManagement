# Solution Task Spec: AC-29

## 1. Task Identity

- Parent Story: AC-13
- Jira Task/Subtask: [`https://nexttoptech.atlassian.net/browse/AC-29`](https://nexttoptech.atlassian.net/browse/AC-29)
- Task Name: BE-03 - Direct Permission Management APIs
- Stack: Backend API
- Status: draft

## 2. Description

Implement APIs for assigning/removing direct permissions and for returning permission-source details to support effective access explainability.

## 3. Goal Of Task

Enable fine-grained access exceptions without requiring role redesign and provide transparent permission source output.

## 4. What Problem This Task Should Be Done For

Role-only access control cannot cover exception cases and lacks source-level transparency.

## 5. AoC (Acceptance of Completion)

- AOC-01: APIs support add/remove direct permissions with consistency checks.
- AOC-02: Permission-source metadata identifies inherited versus direct access.
- AOC-03: Effective-access output aligns with union rule from story acceptance criteria.

## 6. Scope

- In Scope:
  - Direct permission API endpoints
  - Permission source and effective-access response model
  - Validation and conflict handling
- Out Of Scope:
  - Full role administration module
  - Party/company domain and linkage implementation

## 7. Expectations

- Expected outcome:
  - Direct permission APIs with clear effective-access/source metadata consumable by UI and audit flows.
- Success signals:
  - Permission assignment/removal and source attribution are deterministic and testable.

## 8. TDD Coverage

- Unit tests:
  - Permission grant/revoke rules, duplicate handling, and union-calculation tests.
- Integration tests:
  - End-to-end permission mutation and retrieval tests across persistence layer.
- Contract/API tests (if applicable):
  - Response contract tests for source labels and effective-access payload.

## 9. BDD Scenarios

- Scenario 1:
  - Given a user with role-based access
  - When admin grants a direct permission
  - Then effective access includes both sources and source labels are explicit.
- Scenario 2:
  - Given a directly assigned permission
  - When admin removes it
  - Then effective access is recalculated and response reflects the new state.

## 10. DoD (Definition of Done)

- DOD-01: Direct permission APIs and source attribution outputs are complete.
- DOD-02: Union rule and conflict-handling tests pass.
- DOD-03: Mapping to AC-13 AoC and solution plan is documented.
