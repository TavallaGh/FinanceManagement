# Solution Task Spec: AC-28

## 1. Task Identity

- Parent Story: AC-13
- Jira Task/Subtask: [`https://nexttoptech.atlassian.net/browse/AC-28`](https://nexttoptech.atlassian.net/browse/AC-28)
- Task Name: BE-02 - User Lifecycle APIs
- Stack: Backend API
- Status: draft

## 2. Description

Implement user lifecycle endpoints for create, edit, deactivate, soft-delete, and reset-password flows, aligned with security policy and validation rules from AC-13.

## 3. Goal Of Task

Expose reliable API operations that enforce lifecycle rules and support frontend administration workflows.

## 4. What Problem This Task Should Be Done For

Administrators currently lack a complete and consistent API layer to manage user accounts securely.

## 5. AoC (Acceptance of Completion)

- AOC-01: API endpoints cover create/edit/deactivate/delete/reset operations with validation.
- AOC-02: Deactivation immediately blocks authentication and soft-delete preserves history.
- AOC-03: Password changes are restricted to reset flow and comply with security policy.

## 6. Scope

- In Scope:
  - Lifecycle API contract and handlers
  - Validation and policy checks
  - Error responses for key failure cases
- Out Of Scope:
  - UI implementation details
  - Party/company domain and linkage implementation

## 7. Expectations

- Expected outcome:
  - Stable lifecycle APIs that enforce AC-13 business/security rules and are ready for FE integration.
- Success signals:
  - All lifecycle operations behave consistently across normal and failure paths.

## 8. TDD Coverage

- Unit tests:
  - Validation and command-handler tests for create/edit/deactivate/delete/reset operations.
- Integration tests:
  - Endpoint-to-database flow tests including soft-delete and deactivation effects.
- Contract/API tests (if applicable):
  - Request/response contract tests for lifecycle endpoints and error payloads.

## 9. BDD Scenarios

- Scenario 1:
  - Given a valid create-user payload
  - When admin submits the request
  - Then the API creates the user with expected defaults and returns success response.
- Scenario 2:
  - Given an active user account
  - When admin deactivates the account
  - Then login is blocked immediately and state is persisted.

## 10. DoD (Definition of Done)

- DOD-01: All lifecycle endpoints required by AC-13 are implemented and documented.
- DOD-02: Automated tests for happy-path and critical negative-path behaviors pass.
- DOD-03: API behavior is traceable to refinement AoC and solution plan.
