# Solution Task Spec: AC-34

## 1. Task Identity

- Parent Story: AC-13
- Jira Task/Subtask: [`https://nexttoptech.atlassian.net/browse/AC-34`](https://nexttoptech.atlassian.net/browse/AC-34)

- Stack: Frontend + QA
- Status: draft

## 2. Description

Implement an effective-access view that shows permission source per form/resource and add UI tests for critical user-management journeys.

## 3. Goal Of Task

Increase transparency and confidence by making access decisions explainable and verifiable from UI behavior.

## 4. What Problem This Task Should Be Done For

Without source visibility and UI-level tests, permission behavior remains opaque and regression-prone.

## 5. AoC (Acceptance of Completion)

- AOC-01: Effective-access view clearly labels inherited versus direct permission sources.
- AOC-02: Final displayed access reflects union rule of role and direct grants.
- AOC-03: UI test coverage includes key lifecycle and permission scenarios.

## 6. Scope

- In Scope:
  - Access-source visualization UI
  - Effective-access rendering rules
  - UI test scenarios for critical flows
- Out Of Scope:
  - Backend permission engine redesign
  - Party/company domain and linkage implementation

## 7. Expectations

- Expected outcome:
  - Effective-access view clearly explains permission origin and is supported by robust UI tests.
- Success signals:
  - Users can understand why access is granted and regressions are caught early by automated tests.

## 8. TDD Coverage

- Unit tests:
  - Access-source rendering and effective-access computation display logic tests.
- Integration tests:
  - View loading and interaction tests with backend access-source payloads.
- Contract/API tests (if applicable):
  - Frontend contract tests for source and union-access fields.

## 9. BDD Scenarios

- Scenario 1:
  - Given a user with role and direct permissions
  - When admin opens effective-access view
  - Then each permission shows source and final access reflects union rule.
- Scenario 2:
  - Given permission changes were saved
  - When admin refreshes effective-access view
  - Then updated source and access values are displayed.

## 10. DoD (Definition of Done)

- DOD-01: Effective-access/source visualization is complete and understandable.
- DOD-02: UI test suite for critical AC-13 paths passes.
- DOD-03: Accessibility, localization, and layout checks are completed.
