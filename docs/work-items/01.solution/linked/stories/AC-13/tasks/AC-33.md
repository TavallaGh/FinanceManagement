# Solution Task Spec: AC-33

## 1. Task Identity

- Parent Story: AC-13
- Jira Task/Subtask: [`https://nexttoptech.atlassian.net/browse/AC-33`](https://nexttoptech.atlassian.net/browse/AC-33)
- Task Name: FE-03 - Direct Permission UI
- Stack: Frontend
- Status: draft

## 2. Description

Build UI flows for assigning and removing direct permissions and managing exception access independently from role assignments.

## 3. Goal Of Task

Provide administrators with direct control over exception access scenarios.

## 4. What Problem This Task Should Be Done For

Role-only access UI cannot handle operational exceptions efficiently.

## 5. AoC (Acceptance of Completion)

- AOC-01: Admins can assign/remove direct permissions through guided UI interactions.
- AOC-02: UI prevents duplicate or conflicting direct permission operations.
- AOC-03: Permission changes refresh related list/detail views after save.

## 6. Scope

- In Scope:
  - Direct permission management interactions
  - Validation and conflict/error handling
  - Integration with lifecycle and list refresh behavior
- Out Of Scope:
  - Full role administration screens
  - Party/company domain and linkage implementation

## 7. Expectations

- Expected outcome:
  - Direct permission UI allows controlled exception access management with clear user feedback.
- Success signals:
  - Admins can assign/remove direct permissions without ambiguity or duplicate conflicts.

## 8. TDD Coverage

- Unit tests:
  - Permission selection, conflict detection, and state update logic tests.
- Integration tests:
  - UI-to-API permission mutation flow tests and refresh behavior checks.
- Contract/API tests (if applicable):
  - Contract checks for direct permission request/response payloads.

## 9. BDD Scenarios

- Scenario 1:
  - Given admin opens direct-permission panel for a user
  - When admin adds a valid direct permission
  - Then the permission is applied and UI shows updated state.
- Scenario 2:
  - Given a duplicated direct permission attempt
  - When admin submits the change
  - Then operation is blocked with a clear error message.

## 10. DoD (Definition of Done)

- DOD-01: Direct permission UI supports add/remove flows with validations.
- DOD-02: Negative and happy-path tests pass for mutation flows.
- DOD-03: Traceability to AC-13 refinement and plan is documented.
