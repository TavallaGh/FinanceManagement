# Solution Task Spec: AC-32

## 1. Task Identity

- Parent Story: AC-13
- Jira Task/Subtask: [`https://nexttoptech.atlassian.net/browse/AC-32`](https://nexttoptech.atlassian.net/browse/AC-32)
- Task Name: FE-02 - Create/Edit + State Actions UI
- Stack: Frontend
- Status: draft

## 2. Description

Implement user create/edit UI flows with state actions for deactivate, soft-delete, and reset-password initiation, including key validation/error handling.

## 3. Goal Of Task

Enable complete lifecycle administration through a coherent and safe user interface.

## 4. What Problem This Task Should Be Done For

Admins cannot complete lifecycle operations reliably without dedicated create/edit/state-action UX.

## 5. AoC (Acceptance of Completion)

- AOC-01: Create/edit forms support required fields and enforce validations.
- AOC-02: State actions include deactivate and delete behavior aligned with story rules.
- AOC-03: Password changes are routed to reset flow only and key error messages are clear.

## 6. Scope

- In Scope:
  - User create and edit forms
  - Deactivate/delete/reset action controls
  - Validation and error UI states
- Out Of Scope:
  - Permission-source visualization view
  - Party/company domain and linkage implementation

## 7. Expectations

- Expected outcome:
  - Create/edit/state-action UI flows are complete, safe, and aligned to lifecycle rules.
- Success signals:
  - Admins can perform lifecycle actions with clear validation and predictable outcomes.

## 8. TDD Coverage

- Unit tests:
  - Form validation, field rules, and action-state logic tests.
- Integration tests:
  - Submit/update/deactivate/delete/reset-flow integration tests with API responses.
- Contract/API tests (if applicable):
  - Payload contract tests for create/edit/action endpoints.

## 9. BDD Scenarios

- Scenario 1:
  - Given admin opens create-user form
  - When required fields are valid and form is submitted
  - Then the user is created and success state is shown.
- Scenario 2:
  - Given admin chooses reset password action
  - When reset is confirmed
  - Then password change occurs only through reset flow and user feedback is shown.

## 10. DoD (Definition of Done)

- DOD-01: Create/edit/state-action flows satisfy AC-13 acceptance constraints.
- DOD-02: Validation and lifecycle action tests pass.
- DOD-03: Error messages and localization behavior are verified.
