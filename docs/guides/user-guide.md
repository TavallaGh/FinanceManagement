# User Guide (Draft)

## Objective

This guide describes how end users validate delivered features and provide acceptance feedback.

## Review Lifecycle

1. Feature implemented by task/story flow.
2. Technical review completed (`In Review`).
3. Product acceptance performed in `PO Review`.
4. Approved items moved to `Done`.

## What to Verify

- Expected business behavior of the full story.
- No regression in related modules.
- API and UI behavior consistency (if applicable).

## Feedback Format

- Jira key
- Observed behavior
- Expected behavior
- Environment and timestamp
