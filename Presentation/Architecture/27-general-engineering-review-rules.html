Core Principles

DRY SOLID KISS DDD Clean Architecture Async-first

EF Query Rules

  • ()Read query = AsNoTracking
  • Read APIs = DTO/ViewModel projection only
  • Pagination اجباری با Skip/Take
  • Avoid large Include graphs
  • No lazy loading
  • Compiled query برای مسیرهای پرتکرار

EF Write Rules

  • Tracked entities for CUD
  • One SaveChangesAsync per logical transaction
  • No SaveChangesAsync inside loops
  • Use AddRange / bulk operations
  • Use ExecuteUpdateAsync / ExecuteDeleteAsync for bulk mutations

Financial Safety

  • Money type = decimal (مثلا decimal(18,2))
  • Timestamps = UTC (DateTime.UtcNow)
  • Important entities = optimistic concurrency (RowVersion)
  • Financial writes must run in DB transaction
  • Domain validation before persistence (Debit = Credit)

Application Layer Rules

  • Controllers/Endpoints handler-routing only
  • No business logic in controller/endpoint
  • Never expose entities from APIs
  • DB access must be explicit and predictable
  • Index-friendly queries only

Operations

  • Enable DB query logging in development
  • Heavy jobs in background workers
  • Optimize for maintainability and performance
  • Always target production-grade ERP quality
