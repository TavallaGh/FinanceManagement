Localization Architecture and Engineering Guidelines

راهنمای کامل معماری و مهندسی Localization برای Backend، SPA و فرآیند تولید/ریویو کد

Slide 28

Scope (End-to-End)

Backend APIs Frontend SPA Error/Validation Page Titles UI Text Notifications

تمام متن‌های قابل مشاهده کاربر باید Localization شوند. هیچ متن hard-coded کاربرمحور مجاز نیست.

Mandatory Rules

  • Localization کاملا key-based
  • Backend متن ترجمه‌شده برنمی‌گرداند
  • API فقط messageKey برمی‌گرداند
  • Frontend مسئول Resolve و Render ترجمه
  • Entities/Exceptions با متن خام ممنوع

Key Strategy (Strongly Typed)

  • کلیدها در static classes
  • UiKeys / ErrorKeys / ValidationKeys
  • PageKeys / NotificationKeys
  • Compile-time safety + refactor safety
  • Consistency across modules

Naming Convention

  • UI_LOGIN_FORM
  • ERROR_USER_NOT_FOUND
  • VALIDATION_REQUIRED_FIELD
  • PAGE_LOGIN_TITLE
  • NOTIFICATION_PASSWORD_RESET_SUCCESS

Runtime Flow (SPA)

Language Resolution Order

  1. User preference
  2. Tenant configuration
  3. Browser language
  4. Default system language

Storage Options

  • SPA: JSON files (@ngx-translate)
  • apps/erp-web/src/assets/i18n/fa.json
  • apps/erp-web/src/assets/i18n/en.json
  • public/assets/i18n/ (runtime)
  • Language in cookie: erp_app_language (via BFF)
  • Session in HttpOnly cookies (via BFF)

Review Enforcement

  • No hard-coded UI strings
  • No hard-coded error/validation text
  • All user text must use keys
  • Backend must return keys only
  • RTL/LTR: Use logical properties only
