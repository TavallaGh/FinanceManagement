High-Level System Diagram (L1)

نسخه اجرایی مسیر سیستم: User -> Angular SPA -> BFF -> API Host -> Modules -> SQL/Redis

High-Level Architecture

دلیل انتخاب 1: سرعت تحویل MVP

مسیر مستقیم User -> UI -> API Host -> Modules باعث می‌شود تیم بدون پیچیدگی استقرار چندسرویسی، سریع به خروجی قابل استفاده برسد.

دلیل انتخاب 2: کنترل ریسک فنی

قرار گرفتن Identity، Logging، Health و Metrics کنار هسته API از ابتدا، ریسک امنیت، خطای عملیاتی و ابهام پشتیبانی را کاهش می‌دهد.

دلیل انتخاب 3: کارایی و تکامل‌پذیری

ترکیب SQL Server برای OLTP و Redis برای خواندن پرتکرار، هم کارایی MVP را تامین می‌کند و هم مسیر رشد مرحله‌ای معماری را باز نگه می‌دارد.

جمع بندی تصمیم

این دیاگرام نشان می‌دهد انتخاب معماری فعلی یک مصالحه هوشمند است: سادگی استقرار برای فاز MVP، به‌همراه زیرساخت‌های لازم برای امنیت، مشاهده‌پذیری و رشد آینده.
