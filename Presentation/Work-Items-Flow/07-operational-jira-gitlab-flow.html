Operational Flow — Jira ↔ GitLab ↔ Docs

چطور یک Task از نظر عملیاتی اجرا و ردیابی می‌شود

Traceability Core

گام‌های عملیاتی استاندارد

  • Resolve Task از Jira (با key یا URL)
  • اعتبارسنجی Metadata: AoC, DoD, Test Cases, Epic, Fix Version
  • Transition وضعیت مطابق برد (To Do → In Progress → ...)
  • ساخت/بازیابی GitLab Issue و سپس MR (برای Subtask ترتیب اجباری)
  • ثبت لینک‌های دوطرفه Jira Web Links و MR/Issue references
  • ثبت Artifact فاز مربوطه در docs/work-items

قانون مهم Subtask

  • اول GitLab Issue
  • بعد MR
  • Issue/MR باید parent+subtask لینک Jira را داشته باشند

Branch Strategy خلاصه

  • features|bugs|technicals/* روی develop
  • story/* برای promotion به test
  • sprint/* برای promotion به stage
  • hotfix/* مستقیم از main

خروجی نهایی عملیات

  • Jira URL + Status
  • GitLab Issue URL
  • GitLab MR URL
  • Source/Target Branch
