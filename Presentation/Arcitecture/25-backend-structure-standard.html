هر ماژول باید همین الگو را رعایت کند تا مرزبندی دامنه، توسعه Featureها و نگهداری کد در مقیاس بزرگ پایدار بماند.

Handlers Rule

  • Handler یک Facade برای جریان Use Case است
  • Handler می‌تواند چند سرویس مستقل را هماهنگ کند
  • Endpointهای Minimal فقط Request را به Handlers پاس می‌دهند
  • منطق دامنه باید در Rich Domain باقی بماند

Domain Rule

  • ریشه دامنه بر اساس موجودیت کسب‌وکار تعریف می‌شود
  • تفکیک واضح بین Entities / Enums / Aggregates
  • عدم قرار دادن منطق دامنه در Handler یا Endpoint
  • برای ساخت پیچیده از Builder/Factory استفاده شود

Frameworks Rule

  • Result Pattern به‌صورت سراسری الزامی است
  • Result/Option در src/Frameworks نگهداری می‌شود
  • Failure باید شامل ErrorCode + MessageKey باشد

Commons Rule

  • Helpers فقط برای utilityهای داخلی همان ماژول
  • Extensions فقط برای extension methodهای همان ماژول
  • منع اشتراک‌گذاری منطق business بین ماژول‌ها

Benefits

Clear Ownership Faster Onboarding Safer Refactoring Traceable Tests

Current Project Alignment
