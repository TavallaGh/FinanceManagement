Frontend Structure Standard (Blazor)

استاندارد Concrete فرانت‌اند با تفکیک Pages / Clients / Models / Components / Bootstrappers

Slide 26

Frontend Template (Required)

Blazor Root

src/Accounting.Blazor/

  • Layout/
  • Components/ (dynamic reusable)
  • Models/<Module>/Request|Response/
  • Pages/<Module>/<Page>/
  • Clients/<Module>/
  • Bootstrappers/

Page + Client Rule

Pages/<Module>/<Page>/

  • <Page>.razor (markup only)
  • <Page>PageComponent.cs (C# logic)
  • Clients/<Module>/<Page>ApiClient.cs
  • one ApiClient per razor page

Responsibilities

  • .razor فقط Markup و Binding
  • Page component کلاس شبه‌سرویس با منطق C#
  • Api call فقط در ApiClientهای concrete
  • DI فقط در Bootstrappers
  • DI registration با Reflection (ترجیحی)

این ساختار باعث می‌شود منطق UI، قراردادها و تماس با API جدا بمانند و توسعه Featureهای جدید سریع‌تر و امن‌تر انجام شود.

Rule Check

Concrete-only classes One page = one ApiClient RequestModel/ResponseModel Bootstrapper + Reflection DI

Migration Note

  • لایه Service و Interface در فرانت حذف شود
  • Request/Response به Models/<Module>/Request|Response منتقل شود
  • API call مستقیم داخل .razor یا component class مجاز نیست
